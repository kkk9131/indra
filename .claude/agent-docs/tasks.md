# indra - タスク一覧

## コンセプト

ハイブリッド型ローカルAIエージェント:

- **対話モード**: チャットで指示→汎用Agentが直接対応
- **自律モード**: スケジュール→オーケストレーターがサブAgentに委譲→承認→実行

## 自律性ロードマップ

| Level | 名称                             | 状態   | 到達Phase |
| ----- | -------------------------------- | ------ | --------- |
| 0     | 手動（CLI/UIで毎回指示）         | ← 現在 | Phase 1-3 |
| 1     | リアクティブ（イベント駆動）     | -      | Phase 7   |
| 2     | スケジュール駆動（定時自動実行） | -      | Phase 4   |
| 3     | プロアクティブ（機会検知→提案）  | -      | Phase 10  |
| 4     | 学習・適応（結果から戦略改善）   | -      | Phase 11  |

BaseWorkflowパターン:

```
BaseWorkflow<TInput, TOutput, TCheckpoint>
├── execute()     共通ライフサイクル（start → run → complete/fail）
├── retry()       共通リトライ
├── approve()     承認レベル判定
└── abstract run()  各ドメインが実装
```

## 重要度の定義

| レベル | 説明                              |
| ------ | --------------------------------- |
| 必須   | 基盤、他タスクの依存元            |
| 最優先 | 自律実行基盤（次に着手すべき）    |
| 高     | 主要ユースケース（SNS、ニュース） |
| 中     | 外部連携、運用効率化              |
| 横断   | 全機能に関わるログ・分析          |
| 低     | 専門機能（足場）                  |
| 後期   | スケーラビリティ、高度な機能      |
