# X投稿テンプレートライブラリ

## テンプレート一覧

### 1. 質問形式（question）

**アルゴリズムスコア**: 90点

**構造**:

```
[質問フック]

[本文/内容]

[CTA質問]
```

**例**:

```
Claude Codeの新機能もう試した？

今日リリースされた〇〇機能が開発体験を大きく変えそう

■ 自動コード補完
従来の補完とは違いコンテキストを深く理解した上で
次に書くべきコードを予測してくれる

■ コンテキスト理解の向上
ファイル間の依存関係やプロジェクト全体の構造を
把握した上で提案してくれるので精度が段違い

■ 処理速度2倍
レスポンスが高速化されて思考の流れを止めずに
コーディングできるようになった

実際に使ってみると特にリファクタリング作業での
効率が劇的に上がった印象

皆さんはどう活用してる？
```

**最適なケース**:

- エンゲージメント重視
- コミュニティ議論喚起
- 経験共有

---

### 2. ニュース速報形式（breaking）

**アルゴリズムスコア**: 70点

**構造**:

```
[見出し/速報タグ]

[要約（2-3行）]

[詳細リンクまたはCTA]
```

**例**:

```
[速報] Anthropicが新モデルを発表

Claude 4.5 Opusが本日リリース

主要なアップデート内容をまとめる

■ マルチモーダル強化
画像認識精度が向上し複雑な図表やグラフの
解析もより正確に行えるようになった

■ コンテキスト拡張
100K → 200Kトークンへ倍増
大規模なコードベースや長文ドキュメントの
処理がより実用的に

■ 日本語性能
日本語でのニュアンス理解と出力品質が
大幅に向上している

開発者向けのAPIは即日利用可能
具体的な使用感は追ってレポートする
```

**最適なケース**:

- 速報性重視
- 情報の第一報
- フォロワーへの通知

---

### 3. 学び共有形式（learning）

**アルゴリズムスコア**: 85点

**構造**:

```
[学びの宣言]

[具体的な内容（箇条書き）]

[感想または行動喚起]
```

**例**:

```
今日学んだこと

Claude Codeでの開発効率が3倍になったコツを共有する

■ プロンプトは具体的に
「〇〇を修正して」ではなく「〇〇ファイルの△△関数で
□□のエラーが出るので××の処理に変更して」と
具体的に伝えるとほぼ一発で修正してくれる

■ コンテキストは最小限に
関連ファイルを全部渡すより本当に必要な部分だけを
渡す方が精度が上がる
不要な情報はノイズになるよう

■ イテレーションを恐れない
一回で完璧を求めず「80%できたら確認→フィードバック」
のサイクルを回す方が結果的に速い

特に3番目は意識が変わると効果絶大だった

試してみた結果を教えて！
```

**最適なケース**:

- 経験・ナレッジ共有
- コミュニティ貢献
- 信頼性構築

### 4. 実践Tips形式（tip）

**アルゴリズムスコア**: 80点

**構造**:

```
[Tip宣言]

[手順（番号付き）]

[結果または効果]
```

**例**:

```
開発効率を上げるTip

実際に試して効果があったワークフローを紹介する

1. CLIでClaude Codeを起動
   ターミナルから直接起動することで
   IDEとの行き来が不要になる

2. /planで計画を立てる
   いきなりコードを書かせるよりまず設計方針を
   相談することで手戻りが激減する

3. 段階的に実装
   大きな機能は小さなタスクに分解して
   一つずつ確認しながら進める

4. /reviewでコードレビュー
   最後にセルフレビューをかけると
   見落としていた問題を発見できる

このフローに変えてから作業時間が半分になり
バグも大幅に減った

特に2番の計画フェーズが重要
```

**最適なケース**:

- ハウツー共有
- 実用的なアドバイス
- 保存・ブックマーク誘発

**構造**:

```
Claude Codeと会話しながら進められるので
壁打ち相手がいる感覚で開発できた

Next Step
・〇〇機能を追加予定
・ユーザーフィードバックを反映
・パフォーマンス改善

同じような状況の方の参考になれば
```

**最適なケース**:

- 進捗報告
- モチベーション共有
- フォロワーとの繋がり

## テンプレート選択ガイド

| コンテンツタイプ | 推奨テンプレート       |
| ---------------- | ---------------------- |
| ニュース記事     | question > breaking    |
| 技術Tips         | tip > learning         |
| プロダクト発表   | breaking > achievement |
| 経験共有         | learning > question    |

## アルゴリズム最適化のポイント

1. **質問を含める（+20点）**: リプライ誘発
2. **450文字前後（+10点）**: X Premium長文機能を活用した最適な長さ
3. **箇条書き活用（+5点）**: 滞在時間向上
4. **絵文字適度に（+3点）**: 視認性向上
5. **外部リンク単体NG（-15点）**: 避ける

## 長文テンプレート構造

X Premiumの長文機能を活用する場合の基本構造:

```
[質問フック/導入]

[背景説明]

[本文]
■ ポイント1（詳細）
■ ポイント2（詳細）
■ ポイント3（詳細）

[考察/意見]

[CTA質問]
```

目標: 400-500文字
