# Research Report Skill - リサーチレポート作成スキル

## Overview

WebSearchツールを活用して指定されたトピックについて情報を収集し、構造化されたリサーチレポートをMarkdown形式で作成するスキルです。複数の情報源から収集したデータを整理・分析し、信頼性の高いレポートを生成します。

**ワークフロー概要:**

```
[Phase 1] トピック分析・検索クエリ設計
    |
[Phase 2] 情報収集 (WebSearch)
    |
[Phase 3] 情報整理・分析
    |
[Phase 4] レポート作成
    |
[Phase 5] 保存・納品
```

## 使用タイミング

以下のような依頼があった場合にこのスキルを使用する：

- 「〇〇について調べて」
- 「〇〇のリサーチレポートを作成して」
- 「〇〇について検索して教えて」
- 「〇〇の市場調査をして」
- 「〇〇のトレンドを調査して」
- 「〇〇について情報を集めて」
- 「〇〇の比較調査をして」

## 作業ディレクトリ

```
agent-output/research-{YYYYMMDD}-{topic}/
├── work/                     # 作業ファイル
│   ├── search-queries.md     # Phase 1: 検索クエリ一覧
│   ├── raw-sources.md        # Phase 2: 収集した生データ
│   └── analysis.md           # Phase 3: 分析メモ
└── report.md                 # 最終成果物
```

---

## Phase 1: トピック分析・検索クエリ設計

### 目的

ユーザーの調査依頼を分析し、効果的な検索クエリを設計する。

### 手順

1. **ディレクトリ作成**

   ```bash
   mkdir -p agent-output/research-{YYYYMMDD}-{topic}/work
   ```

2. **トピック分析**
   - ユーザーの依頼内容を明確化
   - 調査の範囲・深さを決定
   - 必要な情報カテゴリを特定

3. **検索クエリ設計**

   効果的な検索のため、以下の観点でクエリを設計：
   - メインキーワード + 関連キーワード
   - 異なる角度からのクエリ（定義、事例、比較、最新動向など）
   - 日本語・英語両方のクエリ（必要に応じて）

4. **成果物**

   `work/search-queries.md` に検索クエリ一覧を記録

---

## Phase 2: 情報収集 (WebSearch)

### 目的

WebSearchツールを使用して、複数の観点から情報を収集する。

### 手順

1. **WebSearch実行**

   設計した検索クエリを順次実行：

   ```
   WebSearch: "{検索クエリ1}"
   WebSearch: "{検索クエリ2}"
   WebSearch: "{検索クエリ3}"
   ...
   ```

2. **情報の記録**

   各検索結果から以下を記録：
   - 情報の要約
   - ソースURL
   - 信頼性の評価（公式サイト、ニュースサイト、個人ブログなど）

3. **追加調査**

   初回の検索結果から新たな調査ポイントが見つかった場合、追加で検索を実行

4. **成果物**

   `work/raw-sources.md` に収集した情報を記録

### 検索のベストプラクティス

- 最低3つ以上の異なるクエリで検索
- 公式情報源を優先的に参照
- 最新情報を得るため、日付を含むクエリも検討
- 複数の視点（賛否両論など）を収集

---

## Phase 3: 情報整理・分析

### 目的

収集した情報を整理・分析し、レポートの構成を決定する。

### 手順

1. **情報の分類**
   - 主要な発見事項
   - 詳細情報
   - 補足情報
   - 信頼性の低い情報（除外候補）

2. **重複・矛盾の確認**
   - 複数ソースで一致する情報を優先
   - 矛盾する情報がある場合は両論併記を検討

3. **構成の決定**

   レポートの見出し構成を決定

4. **成果物**

   `work/analysis.md` に分析メモを記録

---

## Phase 4: レポート作成

### 目的

分析結果を基に、構造化されたレポートを作成する。

### レポート構成

```markdown
# {トピック} リサーチレポート

> 作成日: YYYY-MM-DD
> 調査期間: YYYY-MM-DD

## 1. エグゼクティブサマリー

[3-5行で調査結果の要約を記載]

## 2. 調査概要

### 2.1 調査目的

[なぜこの調査を行ったか]

### 2.2 調査範囲

[何を調査対象としたか]

### 2.3 調査方法

[どのように情報を収集したか]

## 3. 調査結果

### 3.1 {主要トピック1}

[詳細な調査結果]

### 3.2 {主要トピック2}

[詳細な調査結果]

### 3.3 {主要トピック3}

[詳細な調査結果]

## 4. 主要な発見・ポイント

- [重要なポイント1]
- [重要なポイント2]
- [重要なポイント3]
- [重要なポイント4]
- [重要なポイント5]

## 5. 結論・推奨事項

[調査から導かれる結論と推奨事項]

## 6. 参考ソース一覧

| No. | ソース名   | URL   | 参照日     |
| --- | ---------- | ----- | ---------- |
| 1   | [ソース名] | [URL] | YYYY-MM-DD |
| 2   | [ソース名] | [URL] | YYYY-MM-DD |
| ... | ...        | ...   | ...        |

---

_このレポートはWebSearchを使用した調査に基づいて作成されました。_
```

### 執筆ガイドライン

- 客観的・中立的な記述を心がける
- 主張には出典を明記する
- 専門用語は必要に応じて説明を付ける
- 箇条書きと段落を適切に使い分ける
- 数値データがある場合は具体的に記載

---

## Phase 5: 保存・納品

### 手順

1. **レポートの最終確認**
   - 誤字脱字のチェック
   - リンク切れがないか確認
   - 構成の一貫性を確認

2. **ファイル保存**

   最終レポートを保存：

   ```
   agent-output/research-{YYYYMMDD}-{topic}/report.md
   ```

## 実行例

```
ユーザー: 「2026年のAIエージェント市場について調べて」

1. [Phase 1] トピック分析
   - 検索クエリ設計:
     - "AIエージェント 市場規模 2026"
     - "AI agent market trends 2026"
     - "AIエージェント 企業 導入事例"
     - "autonomous AI agents comparison"

2. [Phase 2] WebSearchで情報収集
   - 各クエリで検索実行
   - 信頼性の高いソースを優先的に記録

3. [Phase 3] 情報整理
   - 市場規模データの整理
   - 主要プレイヤーの特定
   - トレンドの分析

4. [Phase 4] レポート作成
   - 構造化されたMarkdownレポートを作成

5. [Phase 5] 保存・納品
   - agent-output/research-20261208-ai-agent-market/report.md
```

## カスタマイズポイント

ユーザーの要望に応じて以下をカスタマイズ可能：

- レポートの詳細度（簡潔版 / 詳細版）
- フォーカスする観点（技術面 / ビジネス面 / 比較分析など）
- 出力形式（Markdownのみ / 要約付きなど）
- 追加セクション（SWOT分析、タイムラインなど）
